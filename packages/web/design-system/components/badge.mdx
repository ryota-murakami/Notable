## Usage

```tsx
import { Badge } from '@notable/web/design-system/components'

// Basic usage
<Badge>New</Badge>

// With variant
<Badge variant="secondary">In Progress</Badge>

// With custom styling
<Badge className="bg-blue-100 text-blue-800">Custom</Badge>
```

## Variants

Available badge variants:

- **default**: Primary badge style
- **secondary**: Subtle, secondary style
- **destructive**: For errors or warnings
- **outline**: Border-only style
- **success**: For positive states
- **warning**: For cautionary information

## Common Use Cases

### Status Indicators

```tsx
<Badge variant="success">Active</Badge>
<Badge variant="destructive">Expired</Badge>
<Badge variant="warning">Pending</Badge>
```

### Notification Counts

```tsx
<Badge>99+</Badge>
<Badge variant="destructive">3</Badge>
```

### Tags and Labels

```tsx
<Badge variant="outline">TypeScript</Badge>
<Badge variant="secondary">Draft</Badge>
```

## Best Practices

1. **Keep text short**: Badges should contain 1-2 words maximum
2. **Use semantic variants**: Match the variant to the content meaning
3. **Consistent sizing**: Badges should maintain consistent height across your application
4. **Avoid overuse**: Too many badges can clutter the interface

## Accessibility

- Uses semantic HTML
- Supports keyboard navigation when interactive
- Color contrast meets WCAG guidelines
- Screen reader friendly
