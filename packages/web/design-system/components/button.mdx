# Button

Buttons are interactive elements that trigger actions when clicked. They come in various styles, sizes, and states to
accommodate different use cases throughout the Notable interface.

<Canvas of={ButtonStories.Default} />

## Props

<ArgTypes of={Button} />

## Variants

The Button component supports multiple visual variants to communicate different levels of emphasis and intent.

### Primary

Primary buttons are used for the main call-to-action on a page or in a section.

<Canvas of={ButtonStories.Primary} />

```tsx
<Button variant='primary'>Primary Button</Button>
```

### Secondary

Secondary buttons are used for secondary actions that complement the primary action.

<Canvas of={ButtonStories.Secondary} />

```tsx
<Button variant='secondary'>Secondary Button</Button>
```

### Destructive

Destructive buttons are used for actions that delete data or have significant consequences.

<Canvas of={ButtonStories.Destructive} />

```tsx
<Button variant='destructive'>Delete</Button>
```

### Ghost

Ghost buttons have minimal visual weight and are used for tertiary actions.

<Canvas of={ButtonStories.Ghost} />

```tsx
<Button variant='ghost'>Ghost Button</Button>
```

### Link

Link buttons look like hyperlinks but behave like buttons.

<Canvas of={ButtonStories.Link} />

```tsx
<Button variant='link'>Link Button</Button>
```

### Gradient

Gradient buttons add visual interest for special actions or premium features.

<Canvas of={ButtonStories.Gradient} />

```tsx
<Button variant='gradient'>Gradient Button</Button>
```

## Usage Guidelines

### When to Use

- **Primary Actions**: Use primary buttons for the main action on a page
- **Secondary Actions**: Use secondary buttons for supporting actions
- **Dangerous Actions**: Use destructive buttons for delete or irreversible actions
- **Navigation**: Use link buttons for navigation that doesn't look like traditional links
- **Low Priority**: Use ghost buttons for low-priority actions

### Best Practices

1. **Clear Labels**: Use action-oriented labels that clearly describe what will happen
2. **Button Hierarchy**: Limit primary buttons to one per section to maintain clear hierarchy
3. **Consistent Sizing**: Use consistent button sizes within the same context
4. **Loading States**: Always show loading states for async actions
5. **Disabled States**: Provide tooltips or helper text explaining why a button is disabled

### Accessibility

- All buttons include proper ARIA attributes
- Keyboard navigation is fully supported (Tab, Enter, Space)
- Focus states are clearly visible
- Icon-only buttons must include `aria-label`
- Loading states announce status changes to screen readers

## Keyboard Shortcuts

Buttons support keyboard navigation:

- `Tab` - Focus next button
- `Shift + Tab` - Focus previous button
- `Enter` or `Space` - Activate button
- `Escape` - Remove focus (in some contexts)
